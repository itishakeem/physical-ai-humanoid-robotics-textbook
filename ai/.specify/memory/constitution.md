# Constitution for "Physical AI and Humanoid Robotics"

## 1. Tone and Style
- Educational and formal, but easy to understand
- Avoid jargon unless explained clearly
- Use active voice whenever possible
- Maintain consistency in terminology throughout the book

## 2. Formatting
- Use Markdown with proper headings (`#`, `##`, `###`)
- Include bullet points and numbered lists for clarity
- Use code blocks for examples when necessary
- Include diagrams or figure descriptions where applicable

## 3. Originality and References
- All content must be original and AI-generated
- Cite sources or references when using factual data
- Avoid copying text from external sources

## 4. Chapter Structure
- Each chapter should include:
  - Summary / Introduction
  - Key Concepts
  - Practical Examples
  - Diagrams / Illustrations (described in text)
  - References (if any)
- Chapter length: 500â€“800 words

## 5. Consistency
- Terminology and notation must remain consistent across chapters
- Keep numbering of sections and sub-sections logical
- Maintain a uniform style for headings, examples, and code

## 6. Chatbot Readiness
- Generate content in a structured way so it can be used for RAG-based chatbot
- Use clear paragraphs and separate concepts for easier embeddings
- Include context headers for each section

## 7. Ethics and Accuracy
- Avoid sensitive or biased statements
- Focus on factual accuracy and explain concepts clearly
- Include disclaimers when necessary for speculative or futuristic topics
